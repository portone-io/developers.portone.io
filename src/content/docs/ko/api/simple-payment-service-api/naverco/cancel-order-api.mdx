### 네이버페이 주문형 주문환불 API

<Swagger method="post" path="/payments/{imp_uid}/naver/cancel" baseUrl="https://api.iamport.kr" summary="네이버페이 주문형 주물 환불">
<SwaggerDescription>

(주문형-네이버페이) 네이버페이 상품주문들을 환불처리합니다. 

주문상품의 종류가 여러 개인 경우, 한 건의 포트원 거래고유번호(imp_uid)에 여러 개의 네이버페이 상품주문이 존재하며 일부 또는 전체를 환불처리할 수 있습니다.

`product_order_id` 가 생략될 경우, imp_uid에 해당되는 모든 네이버페이 상품주문들을 환불처리합니다.
환불사유코드는 다음 중 택일이며, 지정하지 않거나 올바르지 않은 값이 전달되면 기본값인 **PRODUCT_UNSATISFIED(서비스 및 상품 불만족)** 을 적용합니다.

여러 건의 상품주문이 환불되는 경우, 일부 주문건만 환불처리에 성공할 수 있습니다.
시도된 상품주문 중 <b>환불처리에 성공한 상품주문</b>에 대한 최종 정보만 제공됩니다.

(NaverProductOrderAnnotation 참조 : 응답 DATA속성은 <b>네이버페이 주문형 포트원 거래고유번호 기준 상품주문 조회 API</b> 의 data 속성과 동일한 구조를 가집니다.)

</SwaggerDescription>

</SwaggerParameter>

### **주요 요청 파라미터 상세 설명**

> **`reason`** <mark stype="color:red;">**\***</mark> <mark style="color:green;">**string**</mark>
> 
> **취소 사유 코드**
> 
> 지정하지 않거나 올바르지 않은 값이 전달되면 기본값인 **PRODUCT_UNSATISFIED(서비스 및 상품 불만족)** 을 적용합니다.
> * SOLD_OUT : 상품 품절
> * DELAYED_DELIVERY : 배송 지연
> * PRODUCT_UNSATISFIED : 서비스 및 상품 불만족 
> * INTENT_CHANGED : 구매 의사 취소 
> * COLOR_AND_SIZE : 색상 및 사이즈 변경 
> * WRONG_ORDER : 다른 상품 잘못 주문 
> * INCORRECT_INFO : 상품 정보 상이
