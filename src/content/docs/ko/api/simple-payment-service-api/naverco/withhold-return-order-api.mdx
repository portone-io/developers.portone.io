### 네이버페이 주문형 상품 반품보류 처리 API

<Swagger method="post" path="/payments/{imp_uid}/naver/request-return" baseUrl="https://api.iamport.kr" summary="네이버페이 상품 반품보류 처리">
<SwaggerDescription>

네이버페이 주문형 상품주문들을 반품보류 처리합니다.

주문상품의 종류가 여러 개인 경우, 한 건의 포트원 거래건(imp_uid)에 여러 개의 네이버페이 상품주문이 존재하며 일부 또는 전체를 반품보류 처리할 수 있습니다.

`product_order_id` 가 생략될 경우, imp_uid에 해당되는 모든 네이버페이 상품주문들을 반품보류 처리합니다.

(NaverProductOrderAnnotation 참조 : 응답 DATA속성은 <b>네이버페이 주문형 포트원 거래고유번호 기준 상품주문 조회 API</b> 의 data 속성과 동일한 구조를 가집니다.)

</SwaggerDescription>

</SwaggerParameter>

<SwaggerParameter name="reason" type="String" required="false">

**반품보류 사유코드**

</SwaggerParameter>

</SwaggerParameter>

### **주요 요청 파라미터 상세 설명**

> 
> **반품보류 사유코드**
> 
> 올바르지 않은 보류사유코드인 경우 기본값 (`ETC`) 를 적용합니다.
