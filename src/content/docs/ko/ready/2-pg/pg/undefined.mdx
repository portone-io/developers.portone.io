## 일반**결제**

<Tabs>
<Tab title="테스트 결제">
#### 테스트 환경 구성방법

1. [포트원 관리자 콘솔 로그인](https://admin.portone.io/integration)
2. 결제연동 탭 클릭
3. 결제 대행사 설정에 테스트 → 네이버페이 결제형 → 네이버페이 결제형 선택후 오른쪽 `+ 추가` 버튼 클릭

<Figure src={image1} />

4. 네이버페이에서 발급받은 연동정보를 입력 후 저장

- **PG상점아이디(파트너 ID)** : 네이버페이 결제형 파트너ID를 입력합니다.
- **ClientID** : 네이버페이측에서 카드사 심사 완료시 가맹점으로 직접 안내됩니다.
- **Client Secret** : 네이버페이측에서 카드사 심사 완료시 가맹점으로 직접 안내됩니다.
- **Chain ID** : 포트원 정산 서비스 및 [V2 모듈](../../../v2-payment/v2) 사용 시 입력 필요합니다. V1 모듈을 사용할 경우엔 SDK 파라미터로 Chain ID를 넘기셔야 합니다. [V1 모듈 네이버페이 연동 가이드 참고](../../../pg/simple/naver)
- **과세구분** : 부가세신고자료용으로 과세 설정을 할 수 있습니다. 미설정/과세/면세/복합과세 중 선택할 수 있습니다.
- **정산용 사이트 키 (API Key)** : 포트원 정산 서비스를 사용하시는 경우, 네이버페이로부터 정산 데이터를 불러오기 위한 사이트 키를 입력하셔야 합니다.

## 정기결제

### 네이버페 정기결제는 결제창 방식만 지원합니다.

<Tabs>
<Tab title="결제창 방식">

보통 일반결제용 채널과 정기결제용 채널은 파트너 ID, Client Id, Client Secret은 동일하고 ChainId가 다릅니다. 포트원 모듈 버전에 따른 정기결제 채널 등록 방식은 아래와 같습니다.

### V1 모듈

V1 모듈의 경우 SDK 파라미터를 통해 Chain Id를 입력하셔야 하기 때문에, 일반결제 채녈과 정기결제 채널을 구분하여 등록할 필요가 없습니다.

### V2 모듈

정기결제용 ChainID를 입력하여 새로운 채널을 생성해주셔야 합니다. 생성한 채널의 이름을 `requestIssueBillingKey` 함수의 `channelName` 파라미터에 입력해주시면 빌링키를 발급하실 수 있습니다.

</Tab>
</Tabs>

<Hint style="info">
**네이버페이 검수란?**

네이버페이 결제형 오픈을 위해서는 네이버페이 검수팀으로 부터 기술검수 통과가 진행되어야 합니다. 해당 검수는 **필수**적으로 진행되어야 하며 해당 검수는 모든 연동이 완료된 이후 당사로 검수요청을 주시면 포트원 1차 검토 이후 이상이 없는 경우 네이버페이로 실 검수 요청하는 프로세스로 진행됩니다.

</Hint>
