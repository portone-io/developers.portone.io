## Deletes a KCP Quick Pay user information.

<Hint style="danger">
  ### Accept setting is required

  You must specify the Accept setting in the HTTP header as follows:

  **Accept : application/json**
</Hint>

<Swagger method="delete" path="/kcpquick/members/{member_id}" baseUrl="https://api.iamport.kr" summary="Deletes a KCP Quick Pay user information.">
  <SwaggerDescription />

  ### Parameters

  #### Path

  <SwaggerParameter name="member_id" type="String">
    **Member ID**
  </SwaggerParameter>

  <SwaggerParameter name="site_code" type="String(80)">
    **KCP Quick Pay site code**
  </SwaggerParameter>

  <SwaggerParameter name="partner_type" type="String(80)">
    **KCP Quick Pay partner type**
  </SwaggerParameter>

  <SwaggerParameter name="partner_subtype" type="String(80)">
    **KCP Quick Pay partner subtype**
  </SwaggerParameter>

  ### Responses

  <SwaggerResponse status="200: OK" description="Success">
    <Tabs>
      <Tab title="KcpQuickMemberResponse">
        **`code`** <mark style="color:red;">**\***</mark> <mark style="color:purple;">**integer**</mark>

        **Response code**

        0: success, Not 0: check the message

        **`message`** <mark style="color:red;">**\***</mark> <mark style="color:green;">**string**</mark>

        **Response message**

        A non-zero code includes a message like 'Invalid payment info'.

        **`response`` `**<mark style="color:red;">**`(KcpQuickMemberAnnotation, optional)`**</mark>
      </Tab>
    </Tabs>

    <Tabs>
      <Tab title="KcpQuickMemberAnnotation">

        **`PG code`**

        **`pg_id`**<mark style="color:red;">**`*`**</mark><mark style="color:green;">**`string`**</mark>

        **`PG Merchant ID`**

        **`inserted`**<mark style="color:red;">**`*`**</mark><mark style="color:orange;">**`integer`**</mark>

        **`User created at (UNIX timestamp)`**

        **`updated`**<mark style="color:red;">**`*`**</mark><mark style="color:orange;">**`integer`**</mark>

        **`User updated at (UNIX timestamp)`**
