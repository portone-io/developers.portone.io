## Saves the payment amount before calling the payment window to block the payment process in case of forgery or falsification of the payment amount when making an authenticated payment.

<Swagger method="post" path="/payments/prepare" baseUrl="https://api.iamport.kr" summary="Saves the payment amount before calling the payment window.">
  <SwaggerDescription>
    If the payment amount of the order saved on the merchant-side is different from the amount passed to IMP.requestpay() for the same merchant

    ---

    uid, the PG payment window call is blocked.
  </SwaggerDescription>

  ### Parameters

  #### Body

  <SwaggerParameter name="merchant_uid" type="String" required="true">
    <mark style="color:red;">
      **Merchant order ID**
    </mark>
  </SwaggerParameter>

  <SwaggerParameter name="amount" type="double" required="true">
    <mark style="color:red;">
      **Payment amount**
    </mark>
  </SwaggerParameter>

  ### Responses

  <SwaggerResponse status="200: OK" description="Success">
    <Tabs>
      <Tab title="PaymentPrepareResponse">
        **`code`** <mark style="color:red;"> **\*** </mark> <mark style="color:purple;"> **integer** </mark>

        **Response code**

        0: success, Not 0: check the message

        **`message`** <mark style="color:red;"> **\*** </mark> <mark style="color:green;"> **string** </mark>

        **Response message**

        A non-zero code includes a message like 'Invalid payment info'.

        <mark style="color:red;">**`response (PaymentPrepareAnnotation, optional)`**</mark>
      </Tab>
    </Tabs>

    <Tabs>

        **`Payment amount`**
      </Tab>
    </Tabs>
  </SwaggerResponse>
