## 주요 업데이트 사항

<prose.h3>
  <PaymentV1 />

  ✔️ 결제 요청 시 파라미터 유효성 검증이 추가되었습니다.
</prose.h3>

결제 요청 시 필수 파라미터인 `amount`와 `merchant_uid`에 대한 유효성 검증 로직이 추가되었습니다. `amount`의 경우 number 형식으로, `merchant_uid`의 경우 string 형식으로 요청하셔야 합니다. 형식이 올바르지 않는 경우 에러가 리턴되며 결제창 호출에 실패하오니 유의해 주시기 바랍니다.

<prose.h3>
  <PaymentV1 /> <PaymentV2 />

  ✔️ 토스페이먼츠(신모듈)을 통한 간편결제 호출 시 POPUP 방식에서 IFRAME으로 변경되었습니다.
</prose.h3>

토스페이먼츠를 통해 카카오페이, 페이코, 삼성페이, Lpay, 토스페이 호출 시 기존에는 POPUP 방식으로 호출되었으나 이제 IFRAME방식으로 변경되어 사용자가 간편결제 창을 닫은 경우에도 정상적으로 실패응답을 확인할 수 있습니다.

<prose.h3>
  <PaymentV1 />

  ✔️ 엑심베이 API 연동 버전이 일부 업데이트 되었습니다.
</prose.h3>

엑심베이를 통해 사용할 수 있는 결제수단에 알리페이플러스가 추가되면서 결제 취소 시 호출되는 API를 최신 버전으로 업데이트 하였습니다.

<prose.h3>
  <PaymentV2 />

  ✔️ 모바일 환경에서 웰컴페이먼츠를 통해 가상계좌 결제 요청시 입금 기한 설정이 가능합니다.

이전에는 모바일 환경에서 웰컴페이먼츠를 통해 가상계좌 결제 요청시 입금 기한 설정이 불가능했으나, 이제 입금 기한 파라미터에 입력된 날짜로 설정할 수 있습니다.
