포트원은 Android, iOS, React Native용 공식 V2 SDK를 제공하고 있으며,
V2 SDK가 제공되지 않는 플랫폼(Flutter 등)에서는 WebView를 직접 구성하여 연동할 수 있습니다.

## 연동원리

모바일 앱에서 결제를 연동하기 전에, 모바일 결제가 어떻게 동작하는지 이해해야 합니다.

### 왜 WebView를 사용하는가?

PG사(카드사, 간편결제 등)는 보안 및 인증 요구사항으로 인해 **웹 기반 결제 UI만 제공**합니다.
카드 번호 입력 폼, 3DS 인증, 간편결제 동의 화면은 모두 웹 페이지로 구현되어 있습니다.

따라서 모바일 앱에서도 네이티브 UI가 아닌 **WebView 안에서 결제 UI를 표시**하게 됩니다.
포트원의 모바일 SDK는 이 WebView를 자동으로 관리하는 얇은 래퍼이며,
SDK 없이 연동할 경우 WebView를 직접 구성해야 합니다.

### 결제 흐름

사용자가 앱에서 결제를 시작하면 다음 과정이 진행됩니다.

```markdown
1. 앱이 WebView를 열고, 포트원 Browser SDK가 포함된 페이지를 로드
2. Browser SDK가 PG 결제 UI를 WebView에 표시
3. 사용자가 카드 정보 입력 / 간편결제 인증 진행
4. (필요시) 외부 앱 실행 — 은행 앱, 카카오페이 등
5. 외부 앱에서 인증 완료 → Deep Link로 앱 복귀
6. PG가 결제 결과와 함께 redirectUrl로 리다이렉트
7. 앱이 리다이렉트를 가로채서 결과 추출
8. 서버에서 결제 검증 후 주문 완료
```

포트원 모바일 SDK는 1\~7단계를 자동으로 처리합니다.

### 웹 결제와의 차이: 리다이렉트 방식

웹 브라우저에서는 `PortOne.requestPayment()`의 Promise 반환값으로 결과를 받을 수 있지만,
모바일 WebView에서는 **리다이렉트 방식**을 사용해야 합니다.

```typescript
PortOne.requestPayment({
  // ... 결제 파라미터
  redirectUrl: "myapp://payment-complete", // 커스텀 URL scheme
});
```

1. `redirectUrl`에 앱의 커스텀 URL scheme을 지정합니다.
2. 결제 완료 시 WebView가 `myapp://payment-complete?paymentId=xxx`로 이동합니다.
3. 앱이 이 URL을 가로채서 쿼리 파라미터에서 결과를 추출합니다.

<Hint style="info">
  모바일 결제에서는 `redirectUrl` 설정이 필수입니다. <br />
  설정하지 않으면 결제 UI가 정상적으로 표시되지 않을 수 있습니다.
</Hint>

### 외부 앱 처리

결제 도중 PG사의 결제 페이지가 카드사 앱카드, ISP 인증, 계좌이체 앱 등을 실행하기 위해 <br />
`kftc-bankpay://`, `ispmobile://`, `intent://` 같은 URL로 WebView를 이동시키는 경우가 있습니다. <br />
앱에서 이러한 URL scheme을 적절히 처리해야 외부 결제 앱이 정상적으로 실행됩니다.

|URL 유형           |예시                |처리 방법                 |
|-------------------|--------------------|--------------------------|
|일반 웹 URL        |`https://...`       |WebView에서 정상 로드     |
|PG/카드사 앱 scheme|`kftc-bankpay://...`|해당 외부 앱 실행         |
|Android Intent     |`intent://...`      |Intent URI 파싱 후 앱 실행|
|커스텀 scheme      |`myapp://...`       |결제 완료 — 결과 추출     |

포트원 모바일 SDK는 HTTP/HTTPS가 아닌 모든 scheme을 감지하여 OS에 전달하는 방식으로 이 분기 처리를 수행합니다.

## SDK를 사용한 연동

포트원은 Android, iOS, React Native용 공식 모바일 SDK를 제공합니다.
SDK가 위 연동원리에서 설명한 WebView 생성, 외부 앱 실행, 결제 결과 수신을 모두 자동으로 처리하므로 가장 간편한 연동 방식입니다.

SDK를 지원하는 플랫폼이라면 SDK를 사용하는 것을 권장합니다.

|플랫폼      |SDK                        |설치 방법            |레퍼런스                                                |
|------------|---------------------------|---------------------|--------------------------------------------------------|
|Android     |`android-sdk`              |JitPack              |[GitHub](https://github.com/portone-io/android-sdk)     |
|iOS         |`PortOneSDK`               |Swift Package Manager|[GitHub](https://github.com/portone-io/ios-sdk)         |
|React Native|`@portone/react-native-sdk`|npm                  |[GitHub](https://github.com/portone-io/react-native-sdk)|

<Hint style="info">
  각 SDK의 설치 및 사용법은 [모바일 SDK 레퍼런스](/sdk/ko/v2-mobile-sdk/readme)를 참고하세요.
  다양한 플랫폼의 연동 예시는 [portone-sample GitHub 저장소](https://github.com/portone-io/portone-sample)에서 확인하실 수 있습니다.
</Hint>

## SDK 없이 직접 연동하기

V2 SDK가 제공되지 않는 플랫폼(Flutter 등)이나 WebView를 직접 제어하고 싶은 경우,
아래 단계를 따라 연동할 수 있습니다.

### Step 1. 의존성 설치

### Step 3. WebView에서 결제 페이지 로드

WebView를 생성하고 포트원 Browser SDK가 포함된 HTML을 로드합니다.
`redirectUrl`에 커스텀 URL scheme을 지정하여 결제 완료 시 앱으로 복귀하도록 합니다.

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <script src="https://cdn.portone.io/v2/browser-sdk.js"></script>
    <script>
      PortOne.requestPayment({
        storeId: "store-...",
        paymentId: "payment-...",
        orderName: "주문명",
        totalAmount: 1000,
        currency: "KRW",
        channelKey: "channel-key-...",
        payMethod: "CARD",
        redirectUrl: "myapp://payment-complete",
      });
    </script>
  </head>
</html>
```

<Hint style="info">
  위 예시는 이해를 돕기 위한 최소 구성입니다.
  실제 연동 시에는 서버에서 `paymentId`를 발급받아 결제 파라미터를 동적으로 전달하세요.
</Hint>

<Hint style="warning">
  WebView에서 JavaScript 실행이 활성화되어 있어야 합니다.
  또한 `redirectUrl`에 사용하는 커스텀 URL scheme(`myapp://`)을
  WebView가 인식할 수 있도록 다음 Step의 URL 네비게이션 인터셉트를 설정해야 합니다.
</Hint>

### Step 4. URL 네비게이션 처리

WebView에서 발생하는 URL 네비게이션을 프로토콜별로 분기 처리해야 합니다.

|URL 프로토콜                              |처리 방법                                  |
|------------------------------------------|-------------------------------------------|
|`http://`, `https://`                     |WebView에서 정상 로드                      |
|커스텀 scheme (`myapp://`)                |결제 완료 — URL 쿼리 파라미터에서 결과 추출|
|`intent://` (Android)                     |Intent URI를 파싱하여 해당 앱 실행         |
|PG/카드사 앱 scheme (`kftc-bankpay://` 등)|해당 외부 앱 실행                          |

<Tabs>
  <Tabs.Tab title="Flutter (Dart)">
    `InAppWebView`에서 `shouldOverrideUrlLoading`을 설정합니다.

    ```dart
    InAppWebView(
      initialSettings: InAppWebViewSettings(
        useShouldOverrideUrlLoading: true,
        resourceCustomSchemes: ["intent"],
      ),
      shouldOverrideUrlLoading: (controller, navigateAction) async {
        final uri = navigateAction.request.url!.rawValue;
        var colon = uri.indexOf(':');
        var protocol = uri.substring(0, colon);

        switch (protocol) {
          case 'http':
          case 'https':
            return NavigationActionPolicy.ALLOW;

          case 'myapp':
            // 결제 완료 - 쿼리 파라미터에서 결과 추출
            final params = navigateAction.request.url!.queryParameters;
            // params['paymentId'], params['code'] 등 활용
            return NavigationActionPolicy.CANCEL;

          case 'intent':
            // Android intent:// URI 파싱 후 앱 실행
            var firstHash = uri.indexOf('#');
            String? scheme;
            for (var param in uri.substring(firstHash + 1).split(';')) {
              var kv = param.split('=');
              if (kv.elementAtOrNull(0) == 'scheme') scheme = kv[1];
            }
            var redirect = '${scheme != null ? '$scheme:' : ''}'
                '${uri.substring(colon + 1, firstHash)}';
            if (await canLaunchUrlString(redirect)) {
              launchUrlString(redirect);
            }
            return NavigationActionPolicy.CANCEL;

          default:
            // 간편결제 앱 등 외부 앱 실행
            if (await canLaunchUrlString(uri)) {
              launchUrlString(uri);
            }
            return NavigationActionPolicy.CANCEL;
        }
      },
    )
    ```
  </Tabs.Tab>

  <Tabs.Tab title="Android (Kotlin)">
    `WebViewClient`의 `shouldOverrideUrlLoading`을 오버라이드합니다.

    ```kotlin
    webView.settings.javaScriptEnabled = true

    webView.webViewClient = object : WebViewClient() {
        override fun shouldOverrideUrlLoading(
            view: WebView, request: WebResourceRequest
        ): Boolean {
            val url = request.url
            return when (url.scheme) {
                "http", "https" -> false
                "myapp" -> {
                    val paymentId = url.getQueryParameter("paymentId")
                    val code = url.getQueryParameter("code")
                    handlePaymentResult(paymentId, code)
                    true
                }
                "intent" -> {
                    val parsed = Intent.parseUri(
                        url.toString(), Intent.URI_INTENT_SCHEME
                    )
                    try {
                        startActivity(parsed)
                    } catch (e: ActivityNotFoundException) {
                        parsed.`package`?.let {
                            startActivity(
                                Intent(Intent.ACTION_VIEW,
                                    Uri.parse("market://details?id=$it"))
                            )
                        }
                    }
                    true
                }
                else -> {
                    try {
                        startActivity(Intent(Intent.ACTION_VIEW, url))
                    } catch (e: ActivityNotFoundException) {
                        // 앱 미설치
                    }
                    true
                }
            }
        }
    }
    ```
  </Tabs.Tab>

  <Tabs.Tab title="iOS (Swift)">
    `WKNavigationDelegate`의 `decidePolicyFor`를 구현합니다.

    ```swift
    func webView(
        _ webView: WKWebView,
        decidePolicyFor navigationAction: WKNavigationAction,
        decisionHandler: @escaping (WKNavigationActionPolicy) -> Void
    ) {
        guard let url = navigationAction.request.url else {
            decisionHandler(.allow)
            return
        }

        switch url.scheme {
        case "http", "https":
            decisionHandler(.allow)
        case "myapp":
            let components = URLComponents(
                url: url, resolvingAgainstBaseURL: false
            )
            let paymentId = components?.queryItems?
                .first(where: { $0.name == "paymentId" })?.value
            handlePaymentResult(paymentId: paymentId)
            decisionHandler(.cancel)
        default:
            // 외부 앱 실행 (간편결제, 은행 앱 등)
            UIApplication.shared.open(url)
            decisionHandler(.cancel)
        }
    }
    ```

### Step 5. 결제 결과 처리

결제가 완료되면 `redirectUrl`로 리다이렉트되며, 쿼리 파라미터에 결제 결과가 포함됩니다.

|키         |설명      |비고   |
|-----------|----------|-------|
|`paymentId`|결제 건 ID|공통   |
|`code`     |오류 코드 |실패 시|
|`message`  |오류 문구 |실패 시|

`code`가 없으면 결제가 정상적으로 완료된 것입니다. `paymentId`를 서버에 전달하여 결제를 검증하세요.

<Hint style="warning">
  클라이언트 측 결제 결과만으로 주문을 확정하지 마세요. 반드시 서버에서 [결제 조회 API](/api/rest-v2/payment#get%20%2Fpayments%2F%7BpaymentId%7D)를 호출하여 결제 상태와 금액을 검증해야 합니다.
</Hint>

서버 측 결제 검증 방법은 [인증 결제 연동하기 - 결제 완료 처리 (서버)](/opi/ko/integration/start/v2/checkout#4-결제-완료-처리-서버)를 참고하세요.
